# The format for defining Splits is as follows:
#
# UNIQUE_ID, SPLIT_NAME | SEGMENT_ID | SEGMENT_ID | ...
#
# UNIQUE_ID -               The unique identifier for this split. It should be different from every other split, and it can be any string
# SPLIT_NAME -              The "fan-name" to give this split.
# SEGMENT_ID -              The unique IDs of the segments to include in this split
# 
# The splits are to be inputted in chronological order starting from the very first split. This is to ensure that each split is numbered appropriately, as they will
# have their numerical ID automatically assigned based on their position in this file, starting from 0. Each segment must also be inputted in its chronological order within each split.
# Each segment MUST be placed at the end of each split line, in chronological order. Each piece of split data MUST ALSO be seperated by a comma,
# as per the format given above. Any format errors in a split will result in that split not being read, followed by a console message in your browser.
# lastly, make sure that a vertical bar character is inserted to seperate the split metadata and the segment information.
